# ISMIR Paper Status

> Last updated: 2026-03-18

## Current State: Blocked on Model Validation

The ISMIR paper revision is paused due to two critical findings that invalidate the reported results. Both must be resolved before submission.

---

## Finding 1: Fold Leaking (SEVERE)

**What:** The CV folds in `data/labels/percepiano/folds.json` leak piece-level data across train/val splits. Segments from the same piece appear in both training and validation sets. In fold 0, 8 of 12 validation pieces also appear in training.

**Impact:** All reported pairwise accuracy numbers are inflated:
- A1-Max ensemble: 80.8% (inflated)
- A1 baseline: 73.9% (inflated)
- Frozen probe: ~50% via comparator, ~76% via regression (inflated)
- All ablation results: inflated

**Root cause:** The current `folds.json` was not generated by the existing piece-stratified fold function (`disentanglement/data/pairwise_dataset.py::create_piece_stratified_folds()`). The correct function exists but wasn't used.

**Fix:** Regenerate `folds.json` using `create_piece_stratified_folds()` with `piece_mapping.json` as input. Then rerun all training (A1-Max sweep + ablation sweep) on clean splits.

**Schema mismatch:** Current `folds.json` is `[{"train": [...], "val": [...], "test": [...]}]` (single split). The `create_piece_stratified_folds()` function outputs `{"fold_0": [...], "fold_1": [...], ...}` (4-fold CV). The training code (`a1_max_sweep.py`) expects the current format. Either the fold generator output needs to be transformed, or the training code needs to be updated.

**Key files:**
- Current folds: `data/labels/percepiano/folds.json`
- Piece mapping: `data/labels/percepiano/piece_mapping.json`
- Correct fold generator: `src/disentanglement/data/pairwise_dataset.py:69-117`
- Training code that reads folds: `src/model_improvement/a1_max_sweep.py:91-94`
- Ablation sweep: `src/model_improvement/ablation_sweep.py`

## Finding 2: Zero Skill-Level Discrimination (SEVERE)

**What:** A1-Max cannot distinguish beginner from professional performances. Evaluated on 26 YouTube Fur Elise recordings across 5 human-labeled skill buckets:

| Bucket | Level | Mean Score | n |
|--------|-------|-----------|---|
| 1 | Beginner | 0.558 | 4 |
| 2 | Early intermediate | 0.566 | 6 |
| 3 | Intermediate | 0.560 | 5 |
| 4 | Advanced | 0.561 | 5 |
| 5 | Professional | 0.565 | 6 |

Total range: 0.008. No dimension shows monotonic skill-level trend.

**Root cause:** PercePiano contains only ~22 advanced-level performers playing 3 Schubert works. The model learned to discriminate within a narrow professional quality band (0.4-0.7) but has no reference for what beginner audio sounds like. Beginner features are out-of-distribution; the regression head defaults to the mean.

**Key files:**
- Skill eval results: `data/evals/skill_eval/ensemble_4fold/fur_elise/results.json`
- Skill eval manifests: documented in `docs/model/01-data.md` (T5 corpus)
- Research timeline: `docs/model/00-research-timeline.md` (lines 34-51)

## Ablation Results (Pre-Fix, Inflated)

These numbers are from the leaking folds and should NOT be published. Preserved for reference only.

| Config | Pairwise (inflated) | R² | Notes |
|--------|--------------------|----|-------|
| Frozen probe (comparator) | 50.7% | 0.47 | Random comparator, meaningless |
| Frozen probe (regression-derived) | 76.2% | 0.47 | From regression scores, still inflated |
| BCE ranking only | 73.0% | -0.26 | |
| BCE + ListMLE | 69.7% | -11.1 | ListMLE without regression anchor destabilizes |
| BCE + ListMLE + CCC | 75.4% | -1.09 | CCC grounds ListMLE |
| Full A1-Max (repro) | 77.5% | 0.12 | Within 2pp of original 78.7% |
| A1-Max ensemble (original) | 80.8% | 0.50 | Deployed model |

**Notable ablation finding (may survive fold fix):** ListMLE alone (69.7%) performs WORSE than BCE-only (73.0%). Adding CCC regression (75.4%) recovers and exceeds BCE-only. This suggests ranking losses need a regression anchor to prevent degenerate solutions. This finding likely holds with clean folds too, since it's about loss dynamics not data leaking.

- Regenerate piece-stratified folds

- Publish whatever numbers come out, with skill-level failure as honest limitation

- Lead with the cautionary tale (inflated accuracy from fold leaking)
- Contribution: proper evaluation methodology for music quality assessment
- Include both inflated and corrected numbers
- ISMIR values methodology papers

### Option C: Fix + A2 multi-tier training (~6-8 weeks) -- PREFERRED
- Fix folds, show accuracy drop (honest baseline)
- Train A2 with YouTube Skill Corpus (T5) + expanded competition data
- Show: "naive single-tier training fails; multi-tier with skill-diverse data recovers"
- Complete narrative arc: problem -> failure -> diagnosis -> fix -> validation
- Strongest ISMIR submission

**Current plan:** Pursue Option C. Option A runs in parallel (clean folds + rerun is prerequisite for both). Option B is the fallback if A2 doesn't converge in time.

## What We Brainstormed (2026-03-16 through 2026-03-18)

### Original Plan (before findings)
- Update papers from frozen MuQ (R²=0.537, 70.3% pairwise) to A1-Max (80.8% pairwise)
- Add systematic ablation (frozen -> LoRA -> loss components)
- Add 7 validation signals (competition, YouTube AMT, cross-soundfont, etc.)
- Lead with pairwise accuracy as primary metric, R² as secondary
- Keep 19 PercePiano dims for comparability, mention 6-dim taxonomy as future work

### Critical discovery: 19-dim vs 6-dim mismatch
- The current papers report 19-dim results. A1-Max was trained on 6 dims.
- These are not directly comparable -- can't put them in the same table.
- Decision: re-run frozen probe on 6 dims for fair baseline.

### Critical discovery: fold leaking
- All pairwise numbers are inflated due to piece-level leaking across train/val.
- The frozen probe's comparator was randomly initialized and frozen, giving ~50% pairwise (meaningless).
- Regression-derived pairwise (from regression scores) was ~76% but still inflated.

### Critical discovery: skill-level failure
- A1-Max scores beginners and professionals identically (0.558 vs 0.565).
- Root cause: PercePiano is 100% advanced-level, 3 Schubert works only.
- Need multi-tier training data (T5 YouTube Skill Corpus) to fix.

### Ablation finding (likely robust)
- ListMLE ranking loss alone HURTS pairwise accuracy (69.7% < 73.0% BCE-only).
- CCC regression loss anchors ListMLE, recovering performance (75.4%).
- Full multi-task (contrastive + invariance) adds another +2pp (77.5%).
- This loss dynamics finding likely survives fold fixing.

### Paper contribution reframing
- Original: "audio encoders achieve 80.8% pairwise accuracy"
- Now: "we found inflated accuracy from fold leaking, fixed it, and show what's real"
- Strongest version (Option C): "single-tier training fails at skill discrimination; multi-tier training with skill-diverse data recovers generalization"

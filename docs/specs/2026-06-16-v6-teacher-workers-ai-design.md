# V6 Teacher on Workers AI (Qwen3-30B-A3B) Design

**Goal:** Run V6 session-end teacher synthesis end-to-end locally with zero Anthropic credits by routing Phase 1 + Phase 2 to Workers AI `@cf/qwen/qwen3-30b-a3b-fp8` through the same authenticated AI Gateway.
**Not in scope:** Streaming chat (`callAnthropicStream`/`teacher-chat-v6`) routed to Workers AI (SSE translation is a separate problem — chat stays on the Anthropic provider, var/auth-fixed only); prompt re-tuning for Qwen; the broader judge/distiller selection in #55; any production deploy.

## Problem
Local V6 teacher synthesis is dead. `apps/api/src/harness/loop/phase1.ts:53` and `phase2.ts:23` build `${env.AI_GATEWAY_TEACHER}/anthropic/v1/messages` with `x-api-key: env.ANTHROPIC_API_KEY`. After the 2026-06-10 gateway unification, `.dev.vars` only defines `AI_GATEWAY_ENDPOINT` + `AI_GATEWAY_TOKEN` — `AI_GATEWAY_TEACHER` and `ANTHROPIC_API_KEY` are undefined, so the harness builds `undefined/anthropic/v1/messages` and throws `v6 phase_error → v6 produced no artifact` (observed live this session on a real Nocturne run: 5 chunks scored by MuQ, AMT transcribed 77 notes, then synthesis failed). Even with the URL fixed, the Anthropic account behind the BYOK gateway is **out of credits** (`400 credit balance too low`, proven via direct gateway curl). The TS tests pass only because they mock `AI_GATEWAY_TEACHER`.

## Solution (from the user's perspective)
Run `just dev` + the local driver on any catalog recording and receive a real `synthesis` artifact — headline (300–500 chars), focus areas, prescribed exercise — generated by Qwen, with no Anthropic spend. Because the chosen model is the teacher **finetune target** (`Qwen/Qwen3-30B-A3B-Instruct-2507`), local runs preview the eventual finetuned teacher, and any weakness is direct signal for `epic:teacher-finetune`.

## Design
Keep Phase 1/2 emitting Anthropic `tool_use`/`tool_result` and translate at the client boundary. A pure **tool-format** module converts an Anthropic Messages request to an OpenAI `chat/completions` request and the OpenAI response back to the Anthropic `{content:[{type:"tool_use"|"text"}], stop_reason}` shape the phases already parse. A **gateway-client** module hides provider routing + gateway URL + auth and delegates to tool-format for the Workers AI provider. `route-model` returns `{provider, model}` from an env toggle (`TEACHER_PROVIDER`, default `workers-ai`), so Anthropic stays selectable the day credits return — a non-destructive switch.

**Trade-offs chosen:**
- Shape-preserving shim over rewriting the phases: keeps the 3-attempt Zod validation-repair loop and the agentic atom loop byte-for-byte unchanged; blast radius is the client layer only.
- Provider toggle over a hard swap: enables a later Sonnet-vs-Qwen A/B and a one-env-var rollback.
- Native function-calling over JSON-mode prompting: Phase 1 is genuinely agentic (multi-atom `tool_choice:auto`); using one mechanism for both phases keeps the shim single-purpose. Proven: `@cf/qwen/qwen3-30b-a3b-fp8` honors forced `tool_choice` through the gateway (HTTP 200, valid `tool_calls`).

**Proven facts (live gateway curl, 2026-06-16 — ground truth for fixtures):**
- `POST ${AI_GATEWAY_ENDPOINT}/workers-ai/v1/chat/completions` with `cf-aig-authorization: Bearer ${AI_GATEWAY_TOKEN}` + `Authorization: Bearer ${CLOUDFLARE_API_TOKEN}` → HTTP 200.
- `@cf/qwen/qwen3-30b-a3b-fp8` + `tool_choice:{type:"function",function:{name}}` → `choices[0].message.tool_calls:[{id,type:"function",function:{name,arguments:"<json string>"}}]`.
- Anthropic path needs `cf-aig-authorization` too (`x-api-key` alone → `AiGatewayError 401`).

## Modules
**tool-format** (`harness/loop/tool-format.ts`)
- Interface: `toOpenAIChatRequest(req)` and `toAnthropicResponse(res)` — pure functions.
- Hides: tool-def mapping (`input_schema`→`function.parameters`), tool_choice mapping (`{type:"auto"}`→`"auto"`, `{type:"tool",name}`→`{type:"function",function:{name}}`), message/content-block mapping (assistant `tool_use[]`→`tool_calls[]`; user `tool_result[]`→`role:"tool"` messages with `tool_call_id`; string content→user message), response mapping (`tool_calls[].function.arguments` JSON-parse→`tool_use.input`, text→text block, `stop_reason` derivation).
- Tested through: the two public functions (Anthropic-shape in → OpenAI-shape out; OpenAI response → Anthropic response). No internal helpers asserted.

**gateway-client** (`harness/loop/gateway-client.ts`)
- Interface: `callModel(env, client, body) => Promise<AnthropicMessageResponse>`.
- Hides: provider branch; gateway URL (`/anthropic/v1/messages` vs `/workers-ai/v1/chat/completions`); auth headers (`cf-aig-authorization` always; `Authorization: Bearer CLOUDFLARE_API_TOKEN` for workers-ai); fetch + non-2xx → `InferenceError`; tool-format delegation for workers-ai.
- Tested through: `callModel` with a mocked `fetch` — assert the Anthropic-shaped return value and the outgoing URL/headers for each provider.

## Verification Architecture
- **Canonical success state:** for a Workers AI response carrying a `write_synthesis_artifact` tool call, `callModel` returns `{content:[{type:"tool_use",name:"write_synthesis_artifact",input:{...}}],stop_reason:"tool_use"}`; end-to-end, the driver emits a `synthesis` event whose `eval_context.artifact` has a 300–500-char headline + `focus_areas` + `prescribed_exercise`.
- **Automated check:** `cd apps/api && bunx vitest run src/harness/loop/tool-format.test.ts src/harness/loop/gateway-client.test.ts src/harness/loop/route-model.test.ts src/harness/loop/phase1.test.ts src/harness/loop/phase2.test.ts src/services/llm.test.ts` — all green.
- **Harness:** no separate Task Group 0 — the tool-format golden fixtures (exact bytes from the proven curl) are built inline as the first vertical tasks. Final manual verification: `just dev` then `CRESCEND_COOKIE=… uv run /tmp/crescend-e2e/drive_cookie.py --wav model/data/evals/practice_eval/nocturne_op9no2/audio/_aySCutsVVQ.wav --piece "nocturne op 9 no 2" --chunks 8` → a real Qwen synthesis artifact.
